# TikTok Native Portal — Agent Instructions

## What This Is

A mobile-friendly web portal where Mani downloads raw slide backgrounds and copies text to manually upload slideshows to TikTok's native app. This preserves TikTok's algorithm preference for native uploads.

**Deployed on Vercel** — auto-deploys on git push.

## Push Workflow: "Push approved slideshows to portal"

When Mani says **"push approved slideshows to portal"** (or similar), run:

```bash
cd /Users/admin/.openclaw/workspace/coding/tiktok-native-portal
node scripts/push-approved.js approved
```

### What the script does:

1. Scans `TikTokGenerator/V1/output/`, `V2/output/`, `V3/output/` for slideshow folders with `meta.json`
2. For each new slideshow (not already in portal):
   - Copies **composite slides** (with text) → `public/slideshows/{id}-slide-{i}.jpg` (for preview)
   - Copies **raw backgrounds** (without text) → `public/raw-images/{id}-slide-{i}.jpg` (for download)
   - Extracts **text per slide** from meta.json (headline + subline for copy buttons)
3. Updates `public/api/slideshows.json`
4. Git commits and pushes → triggers Vercel deploy

### CRITICAL: Raw images source

The raw images come from `{slideshow_folder}/raw/raw_N.jpg`. These are generated by render.py during slideshow creation. They must be:
- **Regular slides**: pure cover-cropped Pinterest background (no text overlays)
- **CTA slides**: background + app screenshot overlay (no text)

If `raw/` folder is missing, the composite (with text) is used as fallback.

### Text extraction from meta.json

The script handles two meta.json formats:

**Structured (preferred)**: `slides` is an array of objects with `type`, `headline`, `text`, etc.
- V1: hook → headline; tip → label + text; cta → label + text
- V2: hook → headline; steps → concatenated step headers/texts; cta → header + app_name + description
- V3: hook → headline; transition → text; methods → joined array; cta → header + app_name + description

**Old/Postiz format**: `slides` is an array of filename strings
- Only `caption` field is available as hook text
- Per-slide text is unavailable

### Other commands

```bash
node scripts/push-approved.js approved 5    # Push only 5 newest
node scripts/push-approved.js push          # Git push only (no new slideshows)
node scripts/push-approved.js clear 20      # Keep only 20 newest, remove rest
node scripts/push-approved.js status        # Show portal status
```

## Portal UI Features

- Swipeable preview carousel (composite images)
- Expandable "Raw Slides & Texte" section per slideshow
- Copy buttons for headline and subline text
- Download individual raw images or all at once
- Downloaded/New badges (tracked in localStorage)
- Delete/Restore (client-side only, localStorage)
